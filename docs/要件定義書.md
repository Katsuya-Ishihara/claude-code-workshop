# チーム向け Todo タスク管理アプリ 要件定義書

> **このファイルは人間が記述する一次情報です。Claude は読み取るだけで編集しません。**
> **ここに書いた内容が、以降の全フェーズの設計・実装の起点になります。**

---

## システム概要

チームメンバーが共同でタスクを管理するための Web アプリケーション。
Todo の作成・担当者アサイン・期限管理・進捗確認を一元化し、
口頭やチャットに頼ったタスク管理からの脱却を目的とする。

---

## 1. プロジェクトの目的

チームメンバーが増え、口頭やチャットでのタスク管理に限界が出てきた。
担当者・期限・進捗を一元管理できるツールを作り、管理コストを削減する。

---

## 2. 対象ユーザー

| ユーザー種別 | 説明 |
|-------------|------|
| 一般メンバー | Todo の作成・更新・完了操作を行う |
| 管理者 | ユーザー管理・カテゴリ管理を行う（フェーズ2） |

---

## 3. 解決したい課題

- 誰が何を担当しているか把握できない
- タスクの期限が抜け落ちる

---

## 4. 開発フェーズ

段階的にリリースし、早期に一般メンバーへ価値を届けることを優先する。

### フェーズ1（初回リリース）- マイルストーン v1.0.0
- **対象ユーザー：** 一般メンバーのみ
- **目標：** 日常のタスク管理を最低限まわせる状態にする

### フェーズ2（次期リリース）- マイルストーン v2.0.0
- **対象ユーザー：** 管理者機能を追加
- **目標：** 運用管理・分析機能を整備し、チーム全体での本格運用を可能にする

---

## 5. 主要機能

| フェーズ | 対象ユーザー | 機能 |
|----------|-------------|------|
| フェーズ1 | 一般メンバー | ユーザー登録・ログイン |
| フェーズ1 | 一般メンバー | Todo の作成・編集・削除・完了 |
| フェーズ1 | 一般メンバー | 担当者のアサイン |

## 6. 対象プラットフォーム

- [x] Webブラウザ（PC）
- [x] Webブラウザ（モバイル対応）※レスポンシブデザイン必須
- [ ] モバイルアプリ（iOS / Android）
- [ ] デスクトップアプリ

---

## 7. 技術的な制約・前提

- フロントエンド：Blazor WASM（.NET 8）
- バックエンド：ASP.NET Core Web API
- データベース：MySQL（社内実績・スキルセットに基づく選定）
- 認証：JWT

---

## 8. デプロイ方針

アプリ開発と同様に、段階的にインフラを成熟させる。

| フェーズ | デプロイ先 | 理由 |
|----------|-----------|------|
| フェーズ1 | AWS EC2 | スタッフの既存スキルで対応可能。まず動く状態を優先する |
| フェーズ2 | AWS ECR（コンテナ化） | GitHub Actions と連携した自動デプロイパイプラインを整備する |

> フェーズ1で機能が安定してからコンテナ化に移行することで、
> アプリとインフラの変更リスクを分離できる。

---

## 9. 非機能要件

| 項目 | 要件 |
|------|------|
| 同時接続数 | 最大50ユーザー |
| レスポンスタイム | 通常操作は1秒以内 |
| 認証 | メールアドレス＋パスワード |
| データ保持 | 削除済みTodoは30日間保持 |

---

## 10. スコープ外（作らないもの）

- モバイルネイティブアプリは対象外
- 外部カレンダー連携は対象外
- 課金・サブスクリプション機能は対象外
